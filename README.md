## Enhancement Three: Databases
The artifact I selected for this enhancement is my CS 340 Animal Shelter dashboard. This artifact is a dashboard application that connects to a MongoDB database containing animal shelter records. The dashboard allows users to view, filter, and analyze animal data based on different rescue categories. It was originally created in CS 340 as part of a client-based project for Grazioso Salvare, where the goal was to build a tool that could help identify animals that may be good candidates for rescue training.
I selected this artifact for my ePortfolio because it demonstrates my ability to work with databases, retrieve information from a large dataset, and connect database operations to a functional user interface. The artifact showcases my skills in MongoDB, Python, data filtering, and dashboard development. Before the enhancement, the dashboard could retrieve and display data, but the database functionality could be improved. For this enhancement, I focused on improving the MongoDB functionality by updating the read method, improving validation, supporting projections, and preparing the database layer for better performance. These changes help reduce unnecessary data retrieval and make the dashboard more efficient and reliable.
This enhancement supports the course outcomes I planned to meet in Module One. It demonstrates my ability to use well-founded and innovative techniques to solve problems, especially by improving how the application handles database queries and data retrieval. It also supports my ability to design and evaluate computing solutions by making the database layer cleaner, safer, and more efficient. At this time, I do not have major updates to my outcome-coverage plans because this enhancement still fits well under the database category. However, the work also connects to software engineering because improving validation and error handling makes the overall application more maintainable.
While enhancing this artifact, I learned more about how important database design is to the performance and reliability of an application. Even if the user interface works, the application can still be improved by making the database operations more efficient. One challenge I faced was updating the read method without breaking the existing dashboard functionality. I had to make sure the method could still handle normal queries while also supporting projections and better validation. This helped me better understand how small changes in the database layer can affect the rest of an application. Overall, this enhancement helped me improve my understanding of MongoDB, data validation, and the importance of writing database code that is secure, efficient, and easier to maintain.
