# CS 499 ePortfolio
## Self Assessment
Throughout my computer science program, I have developed a stronger understanding of software development, problem-solving, databases, algorithms, security, and professional communication. Completing my coursework and building my ePortfolio helped me reflect on my growth as a developer and gave me the opportunity to showcase the skills I have gained across multiple areas of computer science. The program challenged me to work with different programming languages, frameworks, and development tools, including Java, Python, JavaScript, Android Studio, MongoDB, SQL, Node.js, Express, Angular, and cloud-based development environments. These experiences helped me become more adaptable and more confident in my ability to learn new technologies when solving real-world problems.

One of my strengths is my ability to approach problems logically and continue working through challenges even when the solution is not immediately clear. Many of my courses required debugging, testing, and improving existing code, which helped me develop patience and persistence. For example, while working on full stack development, I had to connect a customer-facing web application, an Express backend, a MongoDB database, and an Angular single-page admin application. This gave me experience troubleshooting API requests, database connections, routes, and front-end displays. Outside of my ePortfolio artifacts, projects such as the Travlr Getaways full stack web application helped me understand how different parts of a system work together and why planning, documentation, and testing are important in software engineering.

My coursework also helped shape my professional goals and values. I have become more interested in app development, secure systems, and technology that can make a real impact, especially in areas connected to military or mission-focused systems. My background has helped me value reliability, accountability, and adaptability. In computer science, those values are important because software often supports people, businesses, and organizations that depend on accurate and secure systems. I want to continue developing skills that allow me to build software that is useful, dependable, and secure.

Collaboration and communication were also important parts of the program. Even when projects were completed individually, many assignments required me to think about how software would be used by clients, stakeholders, or team members. For example, the Software Design Document for the Travlr Getaways project required me to explain design constraints, system architecture, and application functionality in a way that a stakeholder could understand. This helped me practice communicating technical ideas clearly instead of only focusing on the code. In a team environment, this type of communication is important because developers need to explain problems, document solutions, ask questions, and make sure everyone understands the purpose of the project.

The program also strengthened my understanding of data structures and algorithms. Through coursework and enhancement planning, I learned how the organization of data can affect performance, readability, and scalability. I practiced using lists, dictionaries, sets, sorting, filtering, and search logic to make applications more efficient. These skills are important because software must do more than simply work; it should also work efficiently and handle larger amounts of data when needed. Understanding algorithms helped me think more carefully about how data moves through a program and how different choices can improve time complexity, memory usage, and user experience.

Software engineering and database development were major parts of my growth as well. I gained experience designing applications that use persistent data, user input, APIs, and structured storage. In my coursework, I worked with databases such as MongoDB, SQL, and Room for Android applications. I learned how to create, read, update, and delete records while also thinking about validation, error handling, and maintainability. These experiences helped me understand that strong software engineering is not just about writing code, but also about organizing that code, documenting it, testing it, and making sure the application can be improved over time.

Security became another important part of my computer science foundation. Through courses involving cryptography and secure software development, I learned about encryption, hashing, authentication, public key cryptography, and secure coding practices. For example, my work with AES and public key cryptography helped me understand how mathematical principles support secure communication. Other coursework, such as secure software analysis, showed me the importance of identifying vulnerabilities, using dependency checks, and protecting sensitive information. These lessons helped me see security as something that should be considered throughout the development process instead of added at the end.

The artifacts included in my ePortfolio work together to demonstrate the full range of my computer science skills. Each artifact represents a different area of the program and shows how I can improve existing software through thoughtful enhancements. One artifact focuses on software engineering and design by improving an Android inventory application with stronger usability, navigation, and maintainability. Another artifact focuses on algorithms and data structures by improving the way dashboard data is processed, filtered, and summarized. The database artifact demonstrates my ability to work with stored data, optimize database interactions, and support more reliable application performance.

Together, these artifacts show my growth as a developer and my ability to apply computer science concepts in practical ways. They demonstrate my experience with front-end design, back-end logic, database management, algorithmic thinking, testing, debugging, and security awareness. More importantly, they show my ability to take an existing project, evaluate its weaknesses, and improve it with clear goals in mind. This ePortfolio represents both the technical skills I have developed and the professional mindset I will bring into the computer science field.
